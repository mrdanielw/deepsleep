# 国产芯片适配评估 — 执行计划

## 一、评估范围

| 维度 | 内容 |
|------|------|
| **场景（5）** | 内容理解、视觉感知、图像生成、视频生成、3D生成 |
| **训练方式（3）** | 预训练、SFT、强化学习 |
| **模型** | 待定（每场景选5个代表性模型） |
| **国产芯片厂商** | 待定 |
| **对比基线** | NVIDIA GPU（A100/H100，具体型号按场景定） |

**评估矩阵：** 5场景 × N模型 × 3训练方式 = 最终评估项数（模型确定后锁定）

## 二、三种训练方式的通用评估指标体系

> 本项目只评估训练能力（预训练、SFT、强化学习），不涉及推理部署。

### 2.0 通用性能指标（三种训练方式共用）

所有评估项均需采集以下**硬件性能指标**，用于国产芯片 vs NV 的直接对比：

| 指标类别 | 具体指标 | 说明 |
|----------|----------|------|
| **吞吐** | samples/sec/GPU 或 tokens/sec/GPU | 按场景选单位（LLM用tokens，CV/生成用samples/steps） |
| **显存占用** | 峰值显存（GB）、显存利用率 | 含模型权重 + 优化器状态 + 激活值 |
| **通信效率** | 多卡AllReduce/AllGather带宽利用率 | 多卡并行场景下必测 |
| **算力利用率** | MFU（Model FLOPs Utilization） | 实际算力 / 理论峰值 |
| **扩展性** | 1卡→2卡→4卡→8卡 线性加速比 | 评估并行策略效率 |
| **稳定性** | 连续训练N小时无故障、loss无异常跳变 | 长稳测试 |

### 2.1 预训练评估指标

| 指标类别 | 具体指标 | 对齐标准（vs NV基线） |
|----------|----------|----------------------|
| **收敛质量** | Training Loss 曲线（loss vs step） | 相同step数下loss偏差 ≤ 5% |
| **收敛质量** | Validation Loss / Perplexity | 相同step数下偏差 ≤ 5% |
| **收敛速度** | 达到目标loss所需step数 | 不超过NV基线的110% |
| **吞吐** | tokens/sec/GPU（LLM）或 samples/sec/GPU（CV/生成） | 达到NV基线的 ≥ 80% |
| **checkpoint一致性** | 相同step保存的权重，加载后loss对齐 | loss偏差 ≤ 1% |

**预训练特有关注点：**
- 混合精度训练（BF16/FP16）的数值稳定性
- 大规模并行（DP/TP/PP/ZeRO）的正确性和效率
- 长时间训练（>24h）的稳定性

### 2.2 SFT（Supervised Fine-Tuning）评估指标

| 指标类别 | 具体指标 | 对齐标准（vs NV基线） |
|----------|----------|----------------------|
| **收敛质量** | SFT Training Loss 曲线 | 相同step数下loss偏差 ≤ 5% |
| **收敛质量** | Validation Loss | 相同step数下偏差 ≤ 5% |
| **下游任务精度** | 任务特定指标（见各场景定义） | 与NV微调结果偏差 ≤ 约定阈值 |
| **吞吐** | samples/sec/GPU | 达到NV基线的 ≥ 80% |
| **过拟合表现** | train loss vs val loss gap | 与NV基线表现一致（gap差异 ≤ 10%） |

**SFT 特有关注点：**
- 不同微调数据规模（1K/10K/100K samples）下的表现
- 多轮对话格式（chat template）处理的正确性（LLM场景）
- 冻结层/可训练层切换的正确性

### 2.3 强化学习（RL）评估指标

> 覆盖 RLHF（PPO）、GRPO、DPO、DDPO 等主流强化学习训练范式。

| 指标类别 | 具体指标 | 对齐标准（vs NV基线） |
|----------|----------|----------------------|
| **Reward** | 平均Reward曲线（reward vs step） | 相同step数下reward偏差 ≤ 5% |
| **Reward** | Reward分布（均值、方差、分位数） | 分布形态一致 |
| **Policy Loss** | Policy gradient loss 曲线 | 趋势一致，数值偏差 ≤ 10% |
| **Value Loss** | Critic/Value function loss（PPO场景） | 趋势一致 |
| **Reward Accuracy** | Reward模型在验证集上的准确率（如适用） | 偏差 ≤ 2% |
| **吞吐** | samples/sec/GPU（含rollout+update全流程） | 达到NV基线的 ≥ 80% |
| **生成质量** | 任务特定指标（见下方场景分解） | 与NV RL训练后结果对齐 |

**强化学习特有关注点：**
- **Rollout阶段**效率：生成/采样阶段的吞吐（RL训练中占比大）
- **多模型协同**：Actor + Critic + Reference Model + Reward Model 的显存管理和调度
- 训练稳定性：RL训练本身波动较大，需评估国产芯片是否引入额外不稳定性

## 三、每个评估项的标准流程（5步）
> 注：以下流程对预训练、SFT、强化学习三种训练方式均适用，"精度"指训练精度（loss对齐、下游指标等），不涉及推理。

| 步骤 | 动作 | 交付物 | 完成标准 | 预估Effort（每模型×场景） |
|------|------|--------|----------|--------------------------|
| **Step 1 NV基线建立** | 在NV GPU上完成该训练方式的完整跑通，记录全部训练指标 | 基线报告（含环境配置、训练脚本、loss曲线、性能数据） | 3次运行结果稳定（CV < 5%） | 10人天 |
| **Step 2 训练适配** | 在目标国产芯片上完成训练流程迁移 | 适配代码 + 环境配置文档 | 训练流程跑通，无报错 | 5人天 |
| **Step 3 精度对齐** | 对比NV基线，调优至训练精度达标 | 精度对比报告（loss曲线、下游指标等） | 各精度指标与NV偏差 ≤ 约定阈值 | 10人天 |
| **Step 4 性能调优** | 算子优化、并行策略调整、显存优化等 | 性能对比报告 + 优化说明 | 训练吞吐达到NV基线 ≥ 约定比例 | 10人天 |
| **Step 5 回归验收** | 全量指标复测，输出最终报告 | 完整评估报告 + 可复现代码包 | 内部复测通过 | 5人天 |

## 四、执行节奏

### 阶段一：前期准备
- [ ] 完成NV基线环境搭建   
- [ ] 确定精度/性能验收阈值 
- [ ] 发布伙伴试水任务对一两个模型进行适配验证

### 阶段二：试水验证
- [ ] 外包伙伴完成试水任务 
- [ ] 内部评审试水结果，确定合作伙伴
- [ ] 确定75项（或实际数量）的优先级排序

### 阶段三：分批执行
-  高优先级评估项，覆盖每个场景至少1个
-  中优先级评估项
-  剩余评估项
- 每批次结束出阶段性汇总报告

### 阶段四：收尾
- 全量数据汇总
- 输出最终评估结论报告
- 代码/文档归档
