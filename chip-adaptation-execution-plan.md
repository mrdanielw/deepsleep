# 国产芯片适配评估 — 执行计划 & 伙伴考核方案

## 一、评估范围

| 维度 | 内容 |
|------|------|
| **场景（5）** | 内容理解、视觉感知、图像生成、视频生成、3D生成 |
| **训练方式（3）** | 预训练、SFT、强化学习 |
| **模型** | 待定（每场景选1-2个代表性模型） |
| **国产芯片厂商** | 待定 |
| **对比基线** | NVIDIA GPU（A100/H100，具体型号按场景定） |

**评估矩阵：** 5场景 × N模型 × 3训练方式 = 最终评估项数（模型确定后锁定）

## 二、每个场景的具体定义

### 场景1：内容理解
- **任务类型：** 文本理解/多模态理解（如文档QA、长文本摘要）
- **代表模型（建议）：** LLM类（如Qwen、GLM系列，待定）
- **评估指标：**
  - 训练：吞吐（tokens/sec/GPU）、收敛曲线（loss vs step对齐）、显存占用
  - 推理：TTFT、输出TPS、Latency P50/P99、吞吐（req/sec）
- **精度指标：** 训练loss对齐（≤5%偏差）；推理输出一致性（BLEU/ROUGE/完全匹配率）
- **评估数据集：** 待定（需包含短/中/长文本，覆盖典型分布）

### 场景2：视觉感知
- **任务类型：** 图像分类、目标检测、语义分割
- **代表模型（建议）：** ViT、YOLO系列、SAM（待定）
- **评估指标：**
  - 训练：吞吐（images/sec/GPU）、收敛曲线、显存占用
  - 推理：Latency P50/P99、吞吐（FPS）
- **精度指标：** mAP/mIoU与NV基线偏差 ≤ X%（X值待定，建议2-5%）
- **评估数据集：** ImageNet（分类）、COCO（检测）、ADE20K（分割）等标准集

### 场景3：图像生成
- **任务类型：** 文生图、图生图
- **代表模型（建议）：** Stable Diffusion系列、DiT（待定）
- **评估指标：**
  - 训练：吞吐（steps/sec/GPU）、收敛曲线、显存占用
  - 推理：单图生成延迟、批量吞吐
- **精度指标：** FID/CLIP Score与NV基线偏差 ≤ X%；人工抽检无明显质量退化
- **评估数据集：** LAION子集/内部数据集（待定）

### 场景4：视频生成
- **任务类型：** 文生视频、图生视频
- **代表模型（建议）：** CogVideoX、Open-Sora系列（待定）
- **评估指标：**
  - 训练：吞吐（steps/sec/GPU）、收敛曲线、显存占用峰值
  - 推理：单视频生成延迟（按分辨率×时长分档）
- **精度指标：** FVD/CLIP Score与NV基线偏差 ≤ X%；人工抽检视频质量
- **评估数据集：** WebVid子集/内部数据集（待定）

### 场景5：3D生成
- **任务类型：** 文本/图像到3D模型生成
- **代表模型（建议）：** TripoSR、InstantMesh系列（待定）
- **评估指标：**
  - 训练：吞吐（steps/sec/GPU）、收敛曲线、显存占用
  - 推理：单模型生成延迟
- **精度指标：** Chamfer Distance/F-Score与NV基线偏差 ≤ X%；人工抽检3D质量
- **评估数据集：** Objaverse子集/ShapeNet（待定）

## 三、每个评估项的标准流程（5步）

| 步骤 | 动作 | 交付物 | 完成标准 |
|------|------|--------|----------|
| **Step 1 NV基线建立** | 在NV GPU上跑通模型，记录全部指标 | 基线报告（含环境配置、脚本、指标数据） | 3次运行结果稳定（CV < 5%） |
| **Step 2 模型适配** | 在目标国产芯片上完成模型迁移 | 适配代码 + 环境配置文档 | 模型能跑通，无报错 |
| **Step 3 精度对齐** | 对比NV基线，调优至精度达标 | 精度对比报告 | 各精度指标与NV偏差 ≤ 约定阈值 |
| **Step 4 性能调优** | 算子优化、并行策略调整等 | 性能对比报告 + 优化说明 | 吞吐/延迟达到NV基线 ≥ 约定比例 |
| **Step 5 回归验收** | 全量指标复测，输出最终报告 | 完整评估报告 + 可复现代码包 | 内部复测通过 |

## 四、执行节奏

### 阶段一：前置准备（第1-2周）
- [ ] 确定参评模型清单（每场景1-2个）
- [ ] 确定参评国产芯片厂商
- [ ] 确定/采集各场景评估数据集
- [ ] 完成NV基线环境搭建
- [ ] 确定精度/性能验收阈值
- [ ] 发布伙伴试水任务（见第五节）

### 阶段二：试水验证（第3-4周）
- [ ] 外包伙伴完成试水任务
- [ ] 内部评审试水结果，确定合作伙伴
- [ ] 确定75项（或实际数量）的优先级排序

### 阶段三：分批执行（第5周起）
- **第一批（第5-8周）：** 高优先级评估项（建议15-20项），覆盖每个场景至少1个
- **第二批（第9-12周）：** 中优先级评估项
- **第三批（第13周+）：** 剩余评估项
- 每批次结束出阶段性汇总报告

### 阶段四：收尾（最后2周）
- 全量数据汇总
- 输出最终评估结论报告
- 代码/文档归档

## 五、伙伴评估方案

### 5.1 试水任务（核心筛选手段）

**不看PPT，看干活。** 给候选伙伴一个标准化试水任务，2周内交付，按结果打分。

**试水任务设计：**
- 选1个场景（建议"视觉感知"，门槛适中、指标明确）
- 选1个模型（如ResNet-50或ViT-Base）
- 要求完成完整5步流程
- 在指定国产芯片上交付

**试水交付物：**
1. NV基线报告
2. 适配后的代码仓库（可复现）
3. 精度对比报告
4. 性能对比报告
5. 遇到的问题及解决方案记录

### 5.2 评分标准

| 评估维度 | 权重 | 评分细则 |
|----------|------|----------|
| **精度达标** | 30% | 偏差≤2%得满分，2-5%得60%，>5%得0 |
| **性能达标** | 25% | 达NV 80%+得满分，60-80%得60%，<60%得0 |
| **代码质量** | 15% | 可复现、文档完整、代码规范 |
| **交付时效** | 15% | 按时交付满分，延期1-3天60%，>3天得0 |
| **问题解决能力** | 15% | 遇到问题的排查记录、沟通响应速度 |

**淘汰线：** 总分 < 60分不进入正式合作

### 5.3 正式合作管理

**周期管理：**
- 周报：每周五提交，含本周完成项、下周计划、阻塞问题
- 周会：每周30min对齐，重点过阻塞和风险
- 阶段验收：每批次结束，内部复测交付物

**质量红线：**
- 交付代码必须可在我方环境复现
- 精度/性能数据造假 → 立即终止合作
- 连续2周无实质进展 → 预警，第3周仍无进展 → 替换

**响应时效：**
- 一般技术问题：24h内回复
- 阻塞问题：4h内响应，升级处理
- 紧急问题：2h内响应

## 六、风险及应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 国产芯片软件栈不成熟，适配成本超预期 | 进度延迟 | 试水阶段提前暴露；预留buffer |
| 模型在国产芯片上精度无法对齐 | 评估结论不可用 | 记录根因，区分芯片问题vs适配问题 |
| 外包团队能力不足 | 交付质量差 | 试水筛选 + 淘汰机制 |
| 评估数据集不统一 | 结果不可比 | 前置准备阶段统一锁定数据集和预处理 |
